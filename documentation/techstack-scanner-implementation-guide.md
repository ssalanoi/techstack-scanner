### 3Ô∏è‚É£ STEP-BY-STEP EXECUTION WORKFLOW

**PHASE 1: Project Initialization (Day 1)**

**Prompt 1 - Create React Frontend:**
```
Create a new React TypeScript project called 'techstack-scanner-frontend' with:
- Vite as build tool
- React Router v6 for routing
- Axios for API calls
- Tailwind CSS for styling
- Basic folder structure with components, pages, layouts, services
```

**Prompt 2 - Create .NET Backend:**
```
Create a new ASP.NET Core Web API project called 'techstack-scanner-backend' with:
- .NET 8
- Controllers for: ScanController, HistoryController
- CORS enabled for localhost:5173
- Swagger/OpenAPI documentation
```

**PHASE 2: Core Functionality (Day 2-3)**

**Prompt 3 - Implement Scanner Service:**
```
Create a C# service that:
- Accepts a folder path
- Scans for: package.json, *.csproj, requirements.txt, Gemfile, pom.xml
- Extracts framework names and versions
- Returns structured data (framework, version, file path, status)
```

**Prompt 4 - Create API Endpoints:**
```
Create REST endpoints:
- POST /api/scan - accepts folder path, returns scan results
- GET /api/scans - returns all previous scans
- GET /api/scans/{id} - returns specific scan
- Use in-memory storage for now
```

**Prompt 5 - Build UI Components:**
```
Create React components:
1. FolderSelector - input for folder path with browse button
2. ScanResults - table/cards displaying technologies
3. TechnologyCard - individual tech item with name, version, status badge
4. ScanHistory - list of previous scans with timestamps
```

**PHASE 3: Layouts & Routing (Day 3-4)**

**Prompt 6 - Implement Layouts:**
```
Create two layouts:
1. MainLayout - header with navigation, main content area, footer
2. AdminLayout - sidebar, header with user menu, content area
Both should use React Router's Outlet component
```

**Prompt 7 - Setup Routing:**
```
Configure React Router with:
- / (Home) - MainLayout
- /scan (Scan page) - MainLayout
- /results/:id (Results detail) - MainLayout
- /history (Scan history) - MainLayout
- /admin/settings (Settings) - AdminLayout
```

**PHASE 4: Integration & Polish (Day 4-5)**

**Prompt 8 - Connect Frontend to Backend:**
```
Create API service layer with:
- Base axios configuration
- Methods for all API endpoints
- Error handling and loading states
- TypeScript interfaces for all DTOs
```

**Prompt 9 - Add State Management:**
```
Implement React Context or Zustand for:
- Current scan state
- Scan history
- User preferences
- Loading/error states
```

**Prompt 10 - Testing & Refinement:**
```
Add basic tests:
- Unit tests for scanner service (C#)
- Component tests for key React components
- Integration test for scan workflow
```

## 8Ô∏è‚É£ TIPS FOR WORKING WITH COPILOT

**Best Practices:**
1. **Be specific** - Include exact technologies, versions, patterns
2. **Provide context** - Mention existing files/structure
3. **Iterate** - Start simple, refine based on results
4. **Review everything** - Never accept code blindly
5. **Break down complex tasks** - Use multiple prompts for complex features

**Common Prompt Patterns:**
- "Create a [component] that [does X] with [technology]"
- "Add [feature] to existing [file] following [pattern]"
- "Refactor [code] to use [better approach]"
- "Write tests for [component] covering [scenarios]"

**When Copilot Struggles:**
- Provide more context about project structure
- Show an example of what you want
- Break the task into smaller pieces
- Manually write a skeleton, then let Copilot fill in
